Root Domain Analysis

▼

Goal

WHOIS, DNS records, ASN discovery, aur reverse WHOIS se organization ke saare registered domains aur IP ranges map karo.

2.1 — WHOIS Lookup
whois / cliHIGH
▼
Purpose: Registrant email + org name milega → reverse WHOIS ke liye use karo. Old domain = forgotten assets. Own nameserver = more attack surface.
Install
sudo apt install whois -y
# Basic WHOIS whois $TARGET | tee ~/recon/$TARGET/notes/whois.txt # Specific fields extract karo whois $TARGET | grep -Ei "registrar|created|expires|name server|org|email|admin" # Kya note karo: # Registrar: → Cloudflare / GoDaddy? # Creation Date: → 2009 ka domain = forgotten assets honge # Name Server: → ns1.target.com = apna DNS server # cloudflare.com = CDN confirm # Registrant Email: → admin@targetcorp.com → save for reverse WHOIS # Registrant Org: → "Target Corp Inc" → exact legal name
2.2 — DNS Records Deep Dive
dig / nslookupCRITICAL
▼
Purpose: TXT records ek gold mine hain — company ki saari third-party services reveal hoti hain. Har service = potential integration = potential attack surface.
Install
sudo apt install dnsutils -y
# All records at once dig $TARGET ANY +noall +answer | tee ~/recon/$TARGET/notes/dns_records.txt # Individual records dig $TARGET A # Main IP dig $TARGET MX # Mail server → mail.target.com type hints dig $TARGET NS # Nameservers dig $TARGET TXT # SPF, DKIM, verification tokens — GOLD dig $TARGET CNAME # Redirects dig $TARGET SOA # Zone info # Zone transfer (rarely works but always try) dig axfr $TARGET @ns1.$TARGET # TXT record examples & what they mean: # "v=spf1 include:sendgrid.net" → SendGrid use karte hain # "google-site-verification=xxx" → Google Search Console # "atlassian-domain-verification" → Jira/Confluence internal tools # "MS=msXXXXXX" → Microsoft 365 / Azure AD
🔥
Zone Transfer Success Saare subdomains ek hi command mein milenge — extremely rare but instant jackpot. Immediately save karo.
💡
TXT Records Har third-party service = alag attack surface. Atlassian found? Jira/Confluence check karo. SendGrid found? Email spoofing test karo.
2.3 — ASN Discovery & IP Range Mapping
bgpview / ipinfo / amassHIGH
▼
Purpose: Company ke saare IP blocks dhundho — yahan forgotten servers, internal tools, aur non-CDN origins milte hain.
# Method 1: IP se ASN dhundho curl -s https://ipinfo.io/$(dig +short $TARGET) | python3 -m json.tool # Output mein "org": "AS12345 Target Corp" milega # Method 2: ASN ke saare IP ranges pull karo curl -s https://api.bgpview.io/asn/ASN_NUMBER/prefixes \ | python3 -c " import json,sys d=json.load(sys.stdin) for p in d['data']['ipv4_prefixes']: print(p['prefix']) " | tee ~/recon/$TARGET/notes/ip_ranges.txt # Method 3: Amass intel amass intel -org "Target Corp" -asn | tee ~/recon/$TARGET/notes/asn.txt # Method 4: bgp.he.net manually check karo in browser # bgp.he.net → search "Target Corp Name"
2.4 — Reverse WHOIS (Related Domains)
viewdns / whoxy / domaintoolsHIGH
▼
Purpose: Registrant email se company ke saare registered domains dhundho — different TLDs, marketing domains, acquired product domains.
# Browser pe manually check karo: viewdns.info/reversewhois/ → registrant email ya org name se whoxy.com/reverse-whois/ → same process domaintools.com/reverse-whois/ → paid but free preview deta hai # Discovered related domains save karo cat > ~/recon/$TARGET/notes/related_domains.txt << EOF $TARGET ← main $TARGET.io ← alternative TLD get$TARGET.com ← marketing domain [add more as found] EOF # Each related domain = run full phase 2-onwards recon again

Subdomain Enumeration — Passive

▼

Goal

Target ke server ko touch kiye bina — internet ke existing databases se saare subdomains collect karo. Completely silent. Stealth level: maximum.

3.1 — Subfinder (40+ Passive Sources)
subfinderCRITICAL
▼
Purpose: 40+ sources (crt.sh, VirusTotal, Shodan, URLScan, SecurityTrails, etc.) se ek saath subdomains collect karo. API keys add karo for 3x more results.
Install
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
# Basic run subfinder -d $TARGET -o subdomains/subfinder.txt # Full power — all sources + recursive subfinder -d $TARGET \ -all \ # saare sources use karo -recursive \ # subdomains ke subdomains bhi -silent \ -o subdomains/subfinder.txt echo "[+] Subfinder: $(wc -l < subdomains/subfinder.txt) subdomains" # API keys add karo for best results # nano ~/.config/subfinder/provider-config.yaml # securitytrails: [key] → securitytrails.com (free) # shodan: [key] → shodan.io (free tier) # virustotal: [key] → virustotal.com (free) # censys: [id:secret] → censys.io (free)
3.2 — Amass (OSINT + Graph Database)
amassCRITICAL
▼
Purpose: Amass ASN→IP→Hostname relationships map karta hai. Subfinder se zyada thorough — OSINT + certificate + DNS sources combine karta hai intelligently.
Install
sudo apt install amass -y
# Passive mode — silent, safe amass enum -passive -d $TARGET -o subdomains/amass_passive.txt # OSINT intelligence mode amass intel -d $TARGET -whois -o subdomains/amass_intel.txt # Verbose — kahan se kya mila dekho amass enum -passive -d $TARGET -v 2>&1 | tee subdomains/amass_verbose.txt
3.3 — Certificate Transparency (crt.sh)
crt.sh / curl + pythonCRITICAL
▼
Purpose: SSL certificates se naming patterns samjho — dev., staging., internal., api-v2. Ye patterns brute force wordlist banane mein kaam aate hain.
# Automated crt.sh query curl -s "https://crt.sh/?q=%25.$TARGET&output=json" \ | python3 -c " import json,sys data=json.load(sys.stdin) names=set() for cert in data: for name in cert['name_value'].split('\n'): name=name.strip().lstrip('*.') if '$TARGET' in name: names.add(name) for n in sorted(names): print(n) " | tee subdomains/crtsh.txt # High-value naming patterns filter karo grep -E "dev\.|staging\.|internal\.|api\.|admin\.|corp\.|test\." subdomains/crtsh.txt # tlsx from ProjectDiscovery (faster alternative) go install github.com/projectdiscovery/tlsx/cmd/tlsx@latest tlsx -san -cn -silent -ro -host $TARGET | anew subdomains/tlsx.txt
Naming patterns like *.corp.target.com, *.internal.target.com in crt.sh = gold. Ye internal naming convention reveal karta hai — brute force wordlist mein add karo immediately.
3.4 — Assetfinder + Wayback Historical Subs
assetfinder / waybackHIGH
▼
Purpose: Assetfinder quick passive enum karta hai. Wayback Machine se historical subdomains milte hain jo ab public CT logs mein nahi hain.
Install
go install github.com/tomnomnom/assetfinder@latest
# Assetfinder assetfinder --subs-only $TARGET | tee subdomains/assetfinder.txt # Wayback Machine historical subdomains curl -s "http://web.archive.org/cdx/search/cdx?url=*.$TARGET&output=text&fl=original&collapse=urlkey" \ | sed -e 's_https*://__' -e "s_/.*__" \ | grep "\.$TARGET$" \ | sort -u | tee subdomains/wayback_subs.txt # Combine ALL passive results cat subdomains/subfinder.txt \ subdomains/amass_passive.txt \ subdomains/assetfinder.txt \ subdomains/crtsh.txt \ subdomains/wayback_subs.txt \ | sort -u | tee subdomains/all_passive.txt echo "[✅] Total passive: $(wc -l < subdomains/all_passive.txt) unique subdomains"
🔥
dev.target.com / staging.target.com found HIGH PRIORITY — seedha manually check karo. Often: weak auth, debug mode on, real prod data, admin panels accessible.
🔥🔥
*.internal.target.com found CRITICAL — DNS resolve karo. Agar public IP hai toh goldmine. Internal tools, admin dashboards accessible ho sakte hain.
📧
mail.target.com / smtp.target.com Mail server found — SPF/DKIM/DMARC check karo. Email spoofing test possible.
🔐
vpn.target.com / remote.target.com VPN gateway — version fingerprint karo, CVEs check karo. Credential stuffing possible.

Subdomain Enumeration — Active (DNS Brute Force)

▼

Goal

Wordlist se potential subdomain names guess karke DNS query bhejte hain. Passive se 3x zyada subdomains nikaal sakta hai — especially those not in public CT logs.

4.1 — PureDNS Brute Force (Best DNS Brute Forcer)
purednsCRITICAL
▼
Purpose: Wildcard DNS automatically filter karta hai — sirf real subdomains deta hai. Jhaddix all.txt = best bug bounty wordlist (9M+ words).
Install
go install github.com/d3mondev/puredns/v2@latest
# Small wordlist — fast start (5000 words) puredns bruteforce \ /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt \ $TARGET \ -r ~/wordlists/resolvers.txt \ -o subdomains/puredns_small.txt # BIG GUN — jhaddix all.txt (best wordlist for bug bounty) puredns bruteforce \ ~/wordlists/jhaddix_all.txt \ $TARGET \ -r ~/wordlists/resolvers.txt \ --wildcard-tests 5 \ -o subdomains/puredns_jhaddix.txt \ --write-wildcards subdomains/wildcards.txt echo "[✅] Brute force: $(wc -l < subdomains/puredns_jhaddix.txt) found" # Resolvers download karo (important for speed) wget https://raw.githubusercontent.com/trickest/resolvers/main/resolvers.txt \ -O ~/wordlists/resolvers.txt # ~7000-10000 resolvers hone chahiye
🔥
Brute force ne naye subdomains diye jo passive ne nahi diye Company apni subdomains public CT logs mein nahi daali → less likely discovered by other hunters → PRIORITY HIGH.
⚠️
Wildcard DNS detect hua *.target.com → sab resolve ho raha hai. puredns automatically handle karta hai — live host probe karke confirm karo.
4.2 — Permutation Scanning (Gotator + Alterx)
gotator / alterxHIGH
▼
Purpose: Found subdomains se smart variations generate karo. api.target.com mila → api-v1, api-dev, api-staging, api-internal automatically try karta hai.
Install
go install github.com/Josue87/gotator@latest && go install github.com/projectdiscovery/alterx/cmd/alterx@latest
# Gotator — permutations from passive results gotator -sub subdomains/all_passive.txt \ -perm ~/wordlists/gotator_perms.txt \ -depth 1 -numbers 3 -silent \ | tee subdomains/permutations.txt # Alterx — smarter permutations (ProjectDiscovery) cat subdomains/all_passive.txt | alterx | tee subdomains/alterx_perms.txt # Resolve the permutations puredns resolve subdomains/permutations.txt \ -r ~/wordlists/resolvers.txt \ -o subdomains/puredns_perms.txt # Example: "api.target.com" → generates: # api-v1, api-v2, api-dev, api-staging, api-old # api-internal, api-test, api2 → any that resolve = NEW FIND 🔥 # Final combine — ALL methods cat subdomains/all_passive.txt \ subdomains/puredns_jhaddix.txt \ subdomains/puredns_perms.txt \ | sort -u > subdomains/all_combined.txt echo "[✅] TOTAL UNIQUE: $(wc -l < subdomains/all_combined.txt)"

🔥🔥

🔥🔥
[200] + "Grafana" Default creds: admin/admin. Data sources mein database credentials. Internal network dashboards visible.

── PHASE 2: ACTIVE SUBDOMAIN ENUM ───────────────── echo "[*] Phase 2: Brute force + permutations..." puredns bruteforce ~/wordlists/jhaddix_all.txt $TARGET \ -r ~/wordlists/resolvers.txt -o subdomains/puredns.txt cat subdomains/all_passive.txt | alterx | puredns resolve \ -r ~/wordlists/resolvers.txt -o subdomains/alterx.txt cat subdomains/all_passive.txt subdomains/puredns.txt subdomains/alterx.txt \ | sort -u > subdomains/all_combined.txt echo "[+] Combined: $(wc -l < subdomains/all_combined.txt) subdomains" # ── PHASE 3: DNS RESOLUTION + LIVE PROBING ───────── echo "[*] Phase 3: DNS resolution + HTTP probing..." dnsx -l subdomains/all_combined.txt -silent -a -cname -o subdomains/resolved.txt httpx -l subdomains/resolved.txt \ -title -status-code -tech-detect -content-length -web-server -follow-redirects -threads 50 \ -o live/httpx_results.txt cat live/httpx_results.txt | cut -d' ' -f1 > live/live_urls.txt echo "[+] Live hosts: $(wc -l < live/live_urls.txt)" # ── PHASE 4: PORT SCAN ────────────────────────────── echo "[*] Phase 4: Port scanning..." cat live/httpx_results.txt | sed 's|https\?://||' | cut -d/ -f1 | sort -u > live/hosts_only.txt naabu -l live/hosts_only.txt \ -p 80,443,8080,8443,8888,8000,9000,9090,3000,5000,4443,6443,2375,9200,5601,3306,5432,6379 \ -silent -o ports/naabu.txt # ── PHASE 5: CONTENT DISCOVERY ────────────────────── echo "[*] Phase 5: Content discovery..." ffuf -u "https://$TARGET/FUZZ" \ -w ~/wordlists/critical_paths.txt \ -mc 200,201,204,301,302,401,403 -t 50 -silent -o content/ffuf.json -of json # ── PHASE 6: URL COLLECTION ───────────────────────── echo "[*] Phase 6: URL collection..." waymore -i $TARGET -mode U -oU urls/waymore.txt 2>/dev/null katana -u https://$TARGET -depth 3 -jc -silent -o urls/katana.txt cat urls/waymore.txt urls/katana.txt | sort -u > urls/MASTER_URLS.txt echo "[+] URLs: $(wc -l < urls/MASTER_URLS.txt)" # ── PHASE 7: JS ANALYSIS ──────────────────────────── echo "[*] Phase 7: JS analysis..." grep -E "\.js$" urls/MASTER_URLS.txt | grep -v ".json" > js/all_js.txt cat js/all_js.txt | while read url; do curl -sk "$url" | jsluice urls 2>/dev/null done | sort -u > endpoints/js_endpoints.txt grep -rE "AKIA[0-9A-Z]{16}|AIza[0-9A-Za-z\-_]{35}|eyJ[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+" \ js/ > secrets/js_secrets.txt # ── PHASE 8: NUCLEI SCAN ──────────────────────────── echo "[*] Phase 8: Nuclei scan..." nuclei -l live/live_urls.txt -s critical,high -silent -o notes/nuclei_crithigh.txt # ── DONE ──────────────────────────────────────────── echo "[✅] RECON COMPLETE FOR: $TARGET" echo "──────────────────────────────────────────" echo "Subdomains: $(wc -l < subdomains/all_combined.txt)" echo "Live hosts: $(wc -l < live/live_urls.txt)" echo "Total URLs: $(wc -l < urls/MASTER_URLS.txt)" echo "Endpoints: $(wc -l < endpoints/js_endpoints.txt)" echo "Secrets: $(wc -l < secrets/js_secrets.txt)" echo "Nuclei hits: $(wc -l < notes/nuclei_crithigh.txt)" echo "──────────────────────────────────────────"
