CRITICAL BROKEN AUTH
TARGET
Verify ANY victim's email by using an OTP generated for attacker's own account (Global OTP flaw).
// EXECUTION CHECKLIST
  • Open two browsers — one for Attacker, one for Victim session
  • Trigger OTP for Victim's email (sent to victim, you can't see it)
  • Start parallel signup with Attacker's own email → receive YOUR OTP
  • Go to Victim's verification page → enter YOUR OTP → intercept in Burp
  • Check: does request contain only {"otp":"123456"} with NO session/email binding?
  • If yes: server validates token globally → VICTIM ACCOUNT VERIFIED by attacker

"Always check if the OTP is 'bound' to the email. If the server only checks if the token exists in its database, the entire authentication is a house of cards."
